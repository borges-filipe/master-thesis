\chapter{Experimental Results\label{cha:chapter5}}
\section{VLM hallucination detection performance}

% Research questions
\begin{formal}
	Are VLMs capable of detecting object hallucinations in Remote Sensing captions?
\end{formal}

\subsubsection{Dataset and evaluation metrics}

\begin{description}
	\item \textbf{RS-Pope}, the Remote Sensing Polling-based Object Probing Evaluation, is a benchmark dataset for evaluating object hallucination in Remote Sensing. Based on POPE's approach, VLMs are queried with a binary question regarding the existence of an object in an image (e.g. "Is there a \textit{tree} in the image?"). The ratio between questions is balanced, 50\% regarding visible and 50\% regarding non-visible objects.
	      The non-existing objects were sampled with three distinct sampling strategies: random sampling, popular sampling based on the top most common objects in the dataset, and adversarial sampling based on object co-occurrence. Additionally, the dataset provides 3 levels of difficulty, which pertains to how many objects are present in a given image: easy, medium and hard. In this experiment the "hard" subset with adversarial sampling was used, with 100 images and a total of 1000 object-probing questions.

	\item \textbf{LID}, the Labelled Illusion Dataset is based on 1804 images from the AeroCaps and VisDrone datasets, with a total of 6668 captions. All captions were manually reviewed, and all hallucinations were tagged at the object level. A total of 217 images resulted in hallucinated objects. For the purpose of this experiment, only the images with hallucinated objects were used, and for each of those images, a binary question was generated regarding the existence of the hallucinated object in the image. To balance the dataset, for each hallucinated object, a non-hallucinated object was sampled from the same image's caption, and a binary question was generated regarding the existence of that object in the image. This resulted in a balanced dataset of 443 binary questions regarding the existence of an object in an image, with 50\% of the questions regarding hallucinated objects and 50\% regarding non-hallucinated objects.

	\item \textbf{Metrics:} The standard binary classification metrics of precision, recall, F1-score, and accuracy were used to evaluate the performance of the models at a fixed decision threshold of 0.5 confidence score. Additionally, the ROC-AUC and Average Precision (AP) were used to evaluate the performance across all possible thresholds. Hallucination detection performance is measured, therefore correctly classifying a hallucination is treated as a True Positive.
\end{description}

\subsection{Binary questions for hallucination detection}

% Research questions
\begin{formal}{}
	Do binary questions on extracted objects successfully detect hallucinations?
\end{formal}


% \textbf{Experimental Setting:} RS-Pope used as a benchmark. RS-Pope has balanced probing questions regarding objects, for example "Is there a building in the image". There are 3 difficulty levels in RS-Pope, here only the hardest one ("adversarial") was used. The questions for which the answer is "no" are treated as hallucinations. As the goal is to detect hallucinations, correctly answering "no" to a question is treated as a True Positive. 

\subsubsection{Implementation details}

The prompt used throughout the experiments is as follows:
\begin{description}
	\item \textbf{Single token answers:} One key modification of the original POPE prompting strategy was made, which is the addition of a direct instruction to answer with "A" or "B". This essentially condenses the model's answer into a single token, which allows for direct comparison of the confidence score of the "A" token (which corresponds to "Yes") and the "B" token (which corresponds to "No"). Having a continuous confidence score allows for more nuanced evaluation of the model's performance and more detailed statistical analysis. Equivalent tokens such as ``A'', ``$\_$A'', `` A'', ``$\dot{G}$A'' were also accepted as valid answers, to account for tokenization differences between models.


	\item \textbf{Prompting:} The full prompt used is:

	      "Is there a \{object\} in the image?

	      A) Yes

	      B) No

	      Letter:"
\end{description}

\subsection{Binary questions vs Multiple Choice Questions}

\begin{formal}
	Are Multiple choice questions with multiple objects more suited for hallucination detection than single object binary questions?
\end{formal}

\subsubsection{Implementation details}
To evaluate multiple choice questions for hallucination detection, the same binary questions were used as a basis, but instead of asking about the existence of a single object, the question was modified to ask about the existence of multiple objects, with one of them being the hallucinated object. To remove positional bias, the same question was asked for all permutations of the objects (capped at 1000). The final confidence scores are the average of all answers.

\begin{description}
	\item \textbf{Prompting:}
	      In The full prompt used is:

	      "What object is visible in the image?

	      A) \{object 1\}

	      B) \{object 2\}

	      \dots

	      C) \{object $n$\}

	      D) None of the above

	      Letter:"

\end{description}

\subsubsection{Effect of different distortion techniques on VCD performance}

\begin{formal}
	Are certain image distortions more suitable for performing VCD on Remote Sensing images?
\end{formal}

\subsubsection{Implementation details}
To test the effect of VCD on hallucination detection, the same binary questions were used. Each question was asked twice, once with the original image and once with the distorted image. The confidence score of the original image was then updated based on the confidence score of the distorted image, using the VCD formula.
\begin{description}
	\item \textbf{Distortion types:}
	      \begin{description}
		      \item \textbf{Diffusion} Like in the original VCD paper, a forward diffusion model was used to introduce noise to the images. The implementation was taken directly from the official VCD GitHub repository.
		      \item \textbf{Gaussian noise} Similar to diffusion, Gaussian noise was added to the images, but without the iterative process of diffusion. The noise was added directly to the pixel values of the image.
		      \item \textbf{Speckle} Speckle was added by multiplying the pixel values of the image with random values drawn from a uniform distribution.
		      \item \textbf{ColorMAE Masking} The ColorMAE masking strategy allows to create different masking patterns that approximate different masking patterns popular for masked auto-encoders, such as block-wise masking, random masking, and grid-wise masking very cheaply. Essentially, different filters are applied to a white noise image and the resulting patterns are used as masks.
	      \end{description}

\end{description}

\subsubsection{Dataset and Evaluation metrics}


\begin{description}
	\item \textbf{Git-10M} is a dataset of 10 million image-caption pairs covering a wide variety of remote sensing imagery. The images span multiple continents and varied scenery. Most images were taken from Google Earth. The captions were generated with GPT-4o. In comparison with other datasets, the captions were generated with very little metadata, such as location or simple keywords such as "airport". The captions therefore seem to be more visually descriptive.
\end{description}



\subsubsection{Semantic entropy for VQA hallucination detection}

\begin{formal}
	Is Semantic Entropy a viable strategy for rejecting hallucinations for Remote Sensing VQA?
\end{formal}


\subsubsection{Implementation details}

\begin{description}
	\item \textbf{Visual Question Generation:} For each image, Qwen3.5 was prompted to extract all the described objects from the caption. For each extracted object, a question was generated for which the object is the correct answer, for example for the object ``rocky terrain'' the question ``What hard, uneven geological formations cover parts of the landscape?'' might be generated.

	\item \textbf{Semantic Entropy Calculation:} The generated questions were then asked the model 11 times. Ten answers are generated with a high temperature of 1, and one answer is generated with a temperature of 0.1 (deemed as the "best" answer). All 11 answers are then checked for bidirectional entailment and semantic groups are formed. From these groups a semantic entropy score is calculated, which is a measure of the diversity of the answers. The low-temperature "best" answer is then compared to the original extracted object to determine correctness.
\end{description}
